
# 소개
  ## *다양한 사람들이 편하게 사용할 수 있는 프로그램을 만들기 위해 팀원들과 토론하는 개발자*
  <br>

  ### **Front End Engineer**

  <br>

  ## 방향성

  ### ● 단순히 소스코드만 찍어내는 것이 아니라 시스템의 아키텍쳐와 설계를 고민하고 나만을 위한 코드가 아닌 팀과 회사, 고객을 위한 개발을 하려고 합니다.
  ### ● 누가 읽어도 이해하기 쉽고 유지보수력을 높이기 위해 가독성 좋고 공통화된 코드를 작성하려는 개발자입니다.
  ### ● 지인들과 개발에 대한 의견을 나누며 다양한 의견을 듣는 것을 좋아하며 피드백을 받고 고민하는 개발자입니다.
  ### ● 웹사이트를 어려워하시는 저희 부모님과 같이 아들한테 안물어봐도 쉽게 사용하실 수 있는 사이트를 만들고 싶어하는 개발자입니다.
  ### ● 다른 사이트들을 돌아다니며 프로젝트에 도입해보는걸 좋아하는 개발자입니다.

  <br>

  ## 장점
  ### ● 협업과 소통 능력
  
  저는 팀원들과의 원활한 소통을 중요하게 여기며, 기술적인 부분을 공유하고 피드백을 주고 받는 것을 적극 지향합니다.  
  팀원 간의 의견 조율을 돕기 위해 노력하며, 직급에 구애받지 않고 객관적인 판단과 논리적 사고를 통해 피드백을 수용하려고 노력합니다.  
  
  또한 스터디 경험을 통해 리더 역할을 수행하며 팀원들 간의 소통과 프로젝트 진행을 원활하게 조율한 경험이 있으며,  
  직장이나 스터디에서도 소규모로 프로젝트를 진행해오면서 기획자 혹은 디자이너 분과 의견을 조율할 때에도  
  더 나은 프로젝트를 만들기 위해 자신의 의견을 적극적으로 이야기하는 편이며,
  요청에 대해서도 개발적으로 가능한지 판단한 후 더 나은 방식을 제안해주신다면 수용하는 자세를 가지고 있습니다.  
  
  
  ### ● 새로운 기술의 습득과 적용
  
  새로운 기술을 습득하는데 빠른편이며, 이전 직장에서 풀스택 개발 경험을 통해 다양한 기술들을 습득하였으며,  
  프론트엔드 개발자로 전향하는 과정에서도 러닝커브를 낮추기 위한 노력을 기울였습니다.  
  
  새로운 기술을 습득할 때는 무작정 학습하는 것이 아니라 프로젝트에서 어떠한 부분이 불편하다 생각이 드는 경우  
  필요한 기술을 선정하고 충분한 학습 후에 적용하려고 합니다.  
  
  지금은 UI와 API 같이 타 직군과 소통이 원활하도록 Figma나 Storybook, Postman 을 어떻게 활용 하면 좋을지 고민하고 있습니다.
  
  ### ● 호기심과 정확한 정보 추구

  저는 호기심이 강하고 팩트 기반의 정보를 추구하는 개발자입니다. 틀린 정보를 싫어하며, 잘못된 정보를 전달하는 것도 꺼립니다.  
  새로운 사실을 알게 될 때에도 검색 결과에 대해 의문을 가지고, 해당 정보가 실제로 사실인지를 판단 후에 적용하는 것을 중요하게 생각합니다.  
  
  기술 블로그나 다른 출처에서도 주장하는 논리가 납득되지 않을 경우, 모든 사람이 동의한다 하더라도  
  정확한 정보를 찾기 위해 책이나 공식 문서를 확인하는 습관을 가지고 있습니다.    
  
  ### ● 스케줄 관리 능력
  
  저는 저의 실력을 객관적으로 판단하는 편이며, 이로 인해 SI 업무를 진행할 때 스케줄에 늦지 않게 개발해왔었습니다.  
  또한 개발 일정이 여유롭지 않은 경우 성격상 작업량을 미리 처리하려는 편이며 매번 마감이 있는 개발을 하는 편이며
  일정을 소화하기 위해 시간을 아끼지 않고, 추후 사업과 연계하여 비용을 생각합니다.
  또한 개발 일정에 따라 포기해야할 부분들과 집중해야하는 부분들을 판단하여 스케줄을 관리해 온 경험이 있습니다.  
  
  ## 이직하려는 이유
  
  저는 입사하기 전까지도 웹 개발을 모르는 상태에서 시작하였었고 솔직하게 신입 때까지는 돌아가기 위한 코드를 먼저 작성해왔었습니다.  
  이전 직장에서는 백엔드 위주의 Full Stack 개발자로 성장하고 싶었으나  
  MPA 환경에서 작업하면서 아래의 클라이언트 관련 문제들에 대해 고민을 하며 시도하던 왔었고  
  오프라인 스터디팀에서 React를 처음 공부하면서 지금껏 고민해왔던 부분들을 이미 다른 개발자들도 고민을 해왔고  
  저와 같은 고민을 했던 사람들이 사용하는 SPA 구조를 활용한 개발자가 되고 싶다는 생각이 들었으며  
  프론트엔드를 할 때가 더 즐겁게 개발을 하고있는 제 모습을 보며 Fronted 개발자로 집중하고 싶어 이직을 결심하였습니다.
